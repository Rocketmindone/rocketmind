# UI Knowledge Sources

Use this file before proposing visual direction. Pick only the entries that match the current task.

## Professional UX/UI Sources

### Nielsen Norman Group

- Best for: task flows, forms, navigation, search, enterprise UX, error prevention
- Use when: the task needs strong usability principles or evidence-backed interaction guidance
- Typical extraction:
  - reduce cognitive load
  - expose system status
  - keep labels specific and user-language based
  - prefer recognition over recall

### Baymard Institute

- Best for: checkout, ecommerce patterns, search, filtering, faceted navigation, form friction
- Use when: the task involves conversion, long forms, filtering, or product discovery
- Typical extraction:
  - keep filter logic predictable
  - avoid hiding critical purchase information
  - reduce ambiguity in validation and next steps

### Material Design

- Best for: state models, adaptive layouts, navigation, accessibility, mobile patterns
- Use when: the task needs structured component behavior or responsive rules
- Typical extraction:
  - keep state transitions consistent
  - define hierarchy with spacing and type before ornament
  - adapt navigation by breakpoint and task frequency

### Apple Human Interface Guidelines

- Best for: touch interactions, mobile ergonomics, content prioritization, status feedback
- Use when: the task is mobile-first or gesture-heavy
- Typical extraction:
  - protect primary tasks from clutter
  - keep feedback immediate and obvious
  - design for thumb reach and interruption

### GOV.UK Design System

- Best for: accessibility, plain language, forms, validation, transactional journeys
- Use when: clarity, trust, or compliance are central
- Typical extraction:
  - one clear purpose per page or section
  - pair errors with fix instructions
  - never rely on color alone

### Atlassian Design System

- Best for: B2B SaaS, dense interfaces, empty states, tables, inline messaging
- Use when: the task is dashboard, admin, or workflow-heavy software
- Typical extraction:
  - help users scan before they commit
  - support progressive disclosure in dense layouts
  - use inline status and guidance to reduce context switches

### Shopify Polaris

- Best for: merchant tools, operational UIs, batch actions, index tables, filters
- Use when: the interface involves list management, permissions, or bulk workflows
- Typical extraction:
  - make bulk actions legible and reversible
  - keep data views configurable but not fragile
  - reveal next steps at the moment of need

## Awwwards Benchmarks

Use these as benchmark references for craft, hierarchy, storytelling, and interaction pacing. Extract principles; do not reproduce compositions.

### Stripe Press / Stripe Product Pages

- Good for: crisp product storytelling, controlled motion, strong visual hierarchy
- Borrow:
  - progressive reveal without blocking comprehension
  - pairing dense information with strong section rhythm

### Linear

- Good for: high-trust SaaS clarity, restrained motion, premium density
- Borrow:
  - disciplined type scale
  - interfaces that feel fast because choices are obvious

### Notion

- Good for: calm productivity layouts, modular information architecture
- Borrow:
  - flexible content framing
  - low-noise surfaces with strong scanning anchors

### Pitch

- Good for: bold product marketing, editorial structure, section pacing
- Borrow:
  - storytelling through alternating density
  - large visual anchors that still preserve CTA clarity

### Spline

- Good for: immersive interactive visuals with clear product framing
- Borrow:
  - expressive hero treatments that still orient the user
  - motion used as explanation, not decoration

### Webflow Enterprise / Webflow Conf Pages

- Good for: conversion-oriented landing pages with premium polish
- Borrow:
  - modular section system
  - balancing brand expression with scannable proof and CTAs

## Selection Heuristic

Choose:
- 2-3 professional sources that best match the task type
- 3-5 benchmark references that fit the desired level of polish and the product category

Then extract:
- one interaction principle
- one hierarchy principle
- one trust or clarity principle
- one responsive or layout principle

If the task already has a mature design system, bias toward system-consistent principles over benchmark novelty.
